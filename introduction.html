Introduction

This is an opinionated guide for contributing effectively.

Goals:

  • make it easy to perform a thorough Code Review
  • create clean, linear and useful commit history
  • make it possible to find the business origin of every commit

Assumptions:

  • the company uses GitHub or similar platform with the concept of Pull Requests (GitHub specific hints are marked with “😺”)
  • branch is being worked on by one person at a time (because in certain cases we’ll rewrite history)
  • the company uses a ticketing system (not a hard requirement, but there are some tips around that area)
